Creating a database

1.     In Object Explorer, connect to an instance of the SQL Server Database Engine and then expand that instance.

2.     Right-click Databases, and then click New Database.

3.     In New Database, enter a database name.

4.     To create the database by accepting all default values, click OK

 

Create Table

  1. In Object Explorer, connect to the instance of Database Engine that contains the database to be modified.
  2. In Object Explorer, expand the Databases node and then expand the database that will contain the new table.
  3. In Object Explorer, right-click the Tables node of your database and then click New Table.
  4. Type column names, choose data types, and choose whether to allow nulls for each column as shown in the following illustration.

  1. To specify more properties for a column, such as identity or computed column values, click the column and in the column properties tab, choose the appropriate properties.

 

  1. To specify a column as a primary key, right-click the column and select Set Primary Key.
  2. To create foreign key relationships, check constraints, or indexes, right-click in the Table Designer pane and select an object from the list as shown in the following illustration.

Filter Records (refer to above comparison operator to filter records):

Below is general command to add a filter. WHERE clause defines that we want to add filter critera. column_name defines which column we want to filter records based on. comparison_operator is one of the equality operator from above table. value is what values we are comparing to, what is stored, in that column.

SELECT column_name,column_name

FROM table_name

WHERE column_name comparison_operator value;

 

Example:  Lets find out who works in department 20 by using equal(=) operator

select * from emp where deptno = 20;

Lets find out who is sales person by using equal(=) operator.

select * from emp where job = 'SALESMAN';

Lets find out who is sales person (=) and has salary more (>) than 1500. We are adding more filters with and operator.

select * from emp where job = 'SALESMAN' and sal > 1500;

 

Lets find out who is sales person or has salary more than 1500. We are adding or operator. This will show all the employees who are either sales person or anybody who has salary more than 1500. Look at the query results. Sales person does not have to have salary greater than 1500 but who is not sales person has to have salary greater than 1500.

select * from emp where job = 'SALESMAN' or sal > 1500;

 

Filter Records based on multiple columns syntax (discussed above):

SELECT column_name,column_name

FROM table_name

WHERE column_name comparison_operator value

AND | OR column_name comparison_operator value;

 

To achieve more readability of the data in our result set, we may have to sort/order the results.

 

Below is the command syntax to order results.

SELECT column_name, column_name

FROM table_name

ORDER BY column_name ASC|DESC, column_name ASC|DESC;

 

Lets find out who is sales person or has salary more than 1500 and sort the query results by employee name. order by clause, followed by column you want to order your result set, is used to sort the result by one or more columns. The order by clause sorts the records in ascending order by default so you do not have to write ASC after the column name.

select * from emp where job = 'SALESMAN' or sal > 1500 order by ename asc;

So far, we have being searching or filter records based on exact values. For example, salary is greater than 1500 or department equals to 20 but what if somebody asks to find out how many employees have names starting with A. This is where we use pattern filter.

Pattern Selection:

SELECT column_name(s)
FROM table_name
WHERE column_name LIKE pattern;

select * from emp where ename like 'A%'

 

In above pattern, there is % sign. This is a wildcard match. What is says is, give me results where first letter is A followed by anything. Below is example where last letter in employee name is D.

select * from emp where ename like '%D'

 

Below example is to find out employees with letter R in their name. Look at the % in the pattern. What it means is, employee name can any characters before and after R or it can start with R or end with R but it should contain R.

select * from emp where ename like '%R%'

 

Range Selection:

SELECT column_name(s)
FROM table_name
WHERE column_name BETWEEN value1 AND value2;

 

Multiple Value Selection:

SELECT column_name(s)
FROM table_name
WHERE column_name IN (value1,value2,...);
