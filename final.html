Constraints

So far, the constraints we know are

Primary Key : Primary key provides two constraints, NOT NULL and Unique. This helps maintain uniqueness and help find data faster. Also, we know that when there is a primary key, you get index behind the scene which actually helps find data faster. This becomes required field and so you have to supply data when you create a new row.

Foreign Key – Foreign key ensures that data created in children tables have referential value in the parent/master table.

Check – Check constraint can be used to ensure data entered in a particular column meets a condition like allowed values for states.

New constraints

Not null – This ensure that a value has to be present when data is created in the table. This values has to be supplied when creating data.

Unique – This ensures that new value entered in a column is unique to what exits in that column in all rows.

Default – This specifies a pre-determined value for a column if value is not supplied during data creation.

Below is the example of how to create constraint in a CREATE TABLE command:

Indexes

Apart from indexes provided by primary key constraint, there will be time when data needs to be queried by using fields that are not part of primary key.

Below is the syntax to create a non-index.

CREATE INDEX index_name
ON table_name (column_name)

You can also create a unique index. This will help store data uniquely in a column or columns and find data fast if it is used right in the query.

CREATE UNIQUE INDEX index_name
ON table_name (column_name)

You have an option to create a composite or compound unique or non-unique index.

CREATE [UNIQUE] INDEX index_name
ON table_name (column_name1,column_name2)

create index idx_foo on dept (deptno);

drop index dept.idx_foo;

create index idx_foo on dept(dname,loc);

drop index dept.idx_foo;

Alter Command

When an object is created in the database and needs to be changed, alter command gives facility to change structure of existing object.

alter table dept_backup add a nvarchar(10);

alter table dept_backup add constraint dept_pk primary key (deptno);

alter table dept_backup drop column a;

 

alter table dept_backup add a nvarchar(10);

alter table dept_backup alter column a nvarchar(15)

 

T-SQL Functions

Like there are built-in functions, SUM, COUNT, UPPER etc, SQL Server allows to create user-defined functions(udf). These functions can be used in

·         T-SQL statements like SELECT, INSERT

·         An application (like website) calling the function

·         Can be used in another function

·         To filter a view

·         To define a check constraint on a column

T-SQL Procedures

·         Accept input parameters and return multiple values in the form of output parameters to the calling procedure or batch.

·         Contain programming statements that perform operations in the database, including calling other procedures.

·         Return a status value to a calling procedure or batch to indicate success or failure (and the reason for failure).

 

--SQL Server Stored Procedure Syntax

CREATE { PROC | PROCEDURE } [schema_name.] procedure_name [ ; number ] 

    [ { @parameter [ type_schema_name. ] data_type }  [ VARYING ] [ = default ] [ OUT | OUTPUT | [READONLY]

 

    ] [ ,...n ]

AS { [ BEGIN ] sql_statement [;] [ ...n ] [ END ] }

[;]

 

Let’s create a procedure that updates employee salary.

create procedure sp_update_sal @empno int, @newsal numeric

as

begin

       update emp

       set sal = @newsal

       where empno = @empno;

end

 

select * from emp where empno = 7369

execute dbo.sp_update_sal 7369,1555

select * from emp where empno = 7369

 

create procedure sp_update_ename @empno int, @newname nvarchar

as

begin

       update emp

       set ename = @newname

       where empno = @empno;

end

 

select * from emp where empno = 7369

execute dbo.sp_update_ename 7369, N'APPLE'

select * from emp where empno = 7369

 

Output parameters

 

create procedure sp_raise_comm @empno int, @raiseby numeric, @newcomm numeric output

as

begin

       update EMP

       set comm = ISNULL(comm,0) + @raiseby

       where empno = @empno;

      

       select @newcomm = comm from emp where empno = @empno;

end

 

 

select * from emp where empno = 7698

declare @newcomm1 numeric

exec sp_raise_comm 7698, 100, @newcomm1 OUTPUT

select @newcomm1
